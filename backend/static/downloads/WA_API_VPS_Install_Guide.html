1. Prepare the VPS

1.1 Update the system

sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential git curl ca-certificates gnupg lsb-release \
                    software-properties-common ufw fail2ban unzip

1.2 Create a dedicated unix user

Never run apps as root. We'll use wa_api and install under /opt/wa_api:

sudo adduser --disabled-password --gecos "" wa_api
sudo mkdir -p /opt/wa_api
sudo chown -R wa_api:wa_api /opt/wa_api

1.3 Firewall

Do NOT open ports 8004 or 3005 to the public.
Nginx will proxy them internally over 127.0.0.1. Only 80 / 443 should be public. 
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw --force enable
sudo ufw status

2. Install runtime dependencies

2.1 Python 3.11

sudo add-apt-repository ppa:deadsnakes/ppa -y
sudo apt update
sudo apt install -y python3.11 python3.11-venv python3.11-dev

2.2 Node.js 20 LTS

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo bash -
sudo apt install -y nodejs
node -v && npm -v
sudo npm install -g yarn

2.3 MongoDB 7 Community Edition

curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc \
     | sudo gpg -o /usr/share/keyrings/mongodb-server-7.0.gpg --dearmor

echo "deb [signed-by=/usr/share/keyrings/mongodb-server-7.0.gpg] https://repo.mongodb.org/apt/ubuntu $(lsb_release -sc)/mongodb-org/7.0 multiverse" \
     | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list

sudo apt update
sudo apt install -y mongodb-org
sudo systemctl enable --now mongod

Mongo binds to 127.0.0.1:27017 by default — safe. WA_API will connect over localhost using its own database (wa_api_db).

2.4 Nginx & Supervisor

sudo apt install -y nginx certbot python3-certbot-nginx supervisor
sudo systemctl enable --now nginx supervisor

3. Get the application code

If you're using Emergent's Save to GitHub, push from the platform first, then clone:

sudo -iu wa_api
cd /opt/wa_api
git clone https://github.com/<your-user>/<wa_api-repo>.git .

Or upload a ZIP via scp from your local machine:

# On your local machine:
scp wa_api.zip wa_api@wa.animitra.in:/opt/wa_api/
# Back on the VPS, as wa_api:
cd /opt/wa_api && unzip wa_api.zip && rm wa_api.zip

Confirm the structure:

ls /opt/wa_api
# backend/  frontend/  wa-sidecar/  ...

4. Configure .env files

4.1 Backend .env

cd /opt/wa_api/backend
nano .env

Paste (regenerate the two secrets with the openssl commands shown):

# ── Database ─────────────────────────────
MONGO_URL="mongodb://127.0.0.1:27017"
DB_NAME="wa_api_db"

# ── Auth / JWT ───────────────────────────
JWT_SECRET="<paste output of: openssl rand -hex 48>"
JWT_ACCESS_MINUTES=60
JWT_REFRESH_DAYS=14
ADMIN_EMAIL="admin@animitra.in"
ADMIN_PASSWORD="ChangeMeAfterFirstLogin!"

# ── Sidecar ──────────────────────────────
WA_SIDECAR_URL="http://127.0.0.1:3005"
SIDECAR_TOKEN="<paste output of: openssl rand -hex 32>"

# ── Public URL (used in emails / QR renders) ──
FRONTEND_ORIGIN="https://wa.animitra.in"
Never ship default secrets.
Generate JWT_SECRET and SIDECAR_TOKEN with: 
openssl rand -hex 48   # for JWT_SECRET
openssl rand -hex 32   # for SIDECAR_TOKEN

4.2 Sidecar .env

cd /opt/wa_api/wa-sidecar
nano .env
PORT=3005
SIDECAR_TOKEN="<same value as backend>"
BACKEND_INTERNAL_URL="http://127.0.0.1:8004"

4.3 Frontend .env (build-time)

cd /opt/wa_api/frontend
nano .env
REACT_APP_BACKEND_URL=https://wa.animitra.in
WDS_SOCKET_PORT=443
This URL is embedded into the compiled JS bundle. If it says localhost, every API call from the browser will fail.

5. Install project dependencies

5.1 Backend (Python)

cd /opt/wa_api/backend
python3.11 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip wheel
pip install -r requirements.txt

5.2 Sidecar (Node)

cd /opt/wa_api/wa-sidecar

cd /opt/wa_api/frontend

6. Supervisor (keep services alive)

Leave the wa_api shell:

exit
sudo mkdir -p /var/log/wa_api && sudo chown wa_api:wa_api /var/log/wa_api

6.1 Backend program

sudo nano /etc/supervisor/conf.d/wa_api_backend.conf
[program:wa_api_backend]
command=/opt/wa_api/backend/.venv/bin/uvicorn server:app --host 127.0.0.1 --port 8004 --workers 2
directory=/opt/wa_api/backend
user=wa_api
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
stdout_logfile=/var/log/wa_api/backend.out.log
stderr_logfile=/var/log/wa_api/backend.err.log
environment=PATH="/opt/wa_api/backend/.venv/bin:/usr/bin:/bin"

6.2 Sidecar program

sudo nano /etc/supervisor/conf.d/wa_api_sidecar.conf
[program:wa_api_sidecar]
command=/usr/bin/node index.js
directory=/opt/wa_api/wa-sidecar
user=wa_api
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
stdout_logfile=/var/log/wa_api/sidecar.out.log
stderr_logfile=/var/log/wa_api/sidecar.err.log

6.3 Start both

sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl status
# Both wa_api_backend and wa_api_sidecar must read RUNNING

Tail the logs to confirm no errors:

sudo tail -n 100 /var/log/wa_api/backend.err.log
sudo tail -n 100 /var/log/wa_api/sidecar.err.log

7. Nginx + HTTPS

7.0 Remove ANY previous config that owns wa.animitra.in
Nginx returns the FIRST matching server_name. If a leftover config (e.g. the old VMP CRM installer) still claims the domain, WA_API will be invisible. 
# See every config that mentions the domain
sudo grep -rli "wa.animitra.in" /etc/nginx/

# See what nginx will actually load for it (read-only)
sudo nginx -T | grep -B2 -A25 "server_name wa.animitra.in"

# Disable the wrong ones (replace with what the grep showed):
sudo rm -f /etc/nginx/sites-enabled/vmp-crm.conf
sudo rm -f /etc/nginx/sites-enabled/default

# Also move the leftover HTML root out of the way (rename, don't delete, until you're sure):
sudo grep -rli "VMP CRM" /var/www /root /home 2>/dev/null
sudo mv /var/www/vmp-installer /var/www/_vmp-installer.old  # example

7.1 Create the WA_API site file

sudo nano /etc/nginx/sites-available/wa_api.conf
server {
    listen 80;
    server_name wa.animitra.in;

    # Serve the compiled React SPA
    root /opt/wa_api/frontend/build;
    index index.html;

    # Everything /api goes to FastAPI on port 8004
    location /api/ {
        proxy_pass         http://127.0.0.1:8004;
        proxy_http_version 1.1;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_set_header   Upgrade           $http_upgrade;
        proxy_set_header   Connection        "upgrade";
        proxy_read_timeout 3600;
        client_max_body_size 60m;
    }

    # SPA fallback
    location / {
        try_files $uri /index.html;
    }

    # Long cache for static assets
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
sudo ln -s /etc/nginx/sites-available/wa_api.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

7.2 Verify nginx is really serving WA_API (before HTTPS!)

Run all three checks BEFORE certbot.
If certbot attaches the cert to the wrong config, you'll be stuck with a cached HTTPS response of the wrong site. 
# a) Config that owns the domain must point to /opt/wa_api and 127.0.0.1:8004
sudo nginx -T | grep -B1 -A20 "server_name wa.animitra.in" \
     | grep -E "root|proxy_pass"
# Expected:
#   root /opt/wa_api/frontend/build;
#   proxy_pass http://127.0.0.1:8004;

# b) Site serves the WA_API landing page
curl -s http://wa.animitra.in/ | grep -o "WA_API\|VMP CRM" | head -1
# Expected:  WA_API      ← if you see "VMP CRM" go back to 7.0

# c) /api is proxied to FastAPI, not falling back to the SPA
curl -s http://wa.animitra.in/api/health
# Expected: {"ok":true,"sidecar":true,"version":"2.0.0"}
# If you see <!DOCTYPE html>, the location /api/ block is missing / not active

7.3 HTTPS with Let's Encrypt

sudo certbot --nginx -d wa.animitra.in \
             --agree-tos -m admin@animitra.in \
             --redirect --non-interactive
sudo systemctl enable --now certbot.timer   # auto-renew twice daily

10. Security hardening

  • Key-only SSH: after adding your public key, edit /etc/ssh/sshd_config → PasswordAuthentication no → sudo systemctl restart ssh.
  • Fail2ban is already installed; default rules cover SSH.
  • MongoDB auth (for shared Mongo): create an admin user, set security.authorization: enabled in /etc/mongod.conf, update MONGO_URL to mongodb://user:pass@127.0.0.1:27017/wa_api_db.
  • Unattended upgrades: sudo apt install -y unattended-upgrades.
  • Rotate JWT_SECRET & SIDECAR_TOKEN every 90 days — restart both supervisor programs after.

11. Troubleshooting

Backend won't start

sudo tail -n 200 /var/log/wa_api/backend.err.log
  • Wrong Python → recreate the venv with python3.11 -m venv .venv.
  • Mongo not running → sudo systemctl status mongod.

Sidecar won't start

sudo tail -n 200 /var/log/wa_api/sidecar.err.log
  • Port 3005 in use → sudo ss -tlnp | grep 3005.
  • SIDECAR_TOKEN mismatch → must be identical in both .env files.

QR never renders

The sidecar can't reach WhatsApp servers. Check curl -I https://web.whatsapp.com from the VPS. Some low-cost VPS providers block outbound WebSocket — contact support.

CRM sees "not connected" even after scanning

curl -H "Authorization: Bearer <KEY>" \
     https://wa.animitra.in/api/v1/sessions/<slug>/status

If sidecar_reachable: false, your backend can't reach the sidecar — check supervisor status and WA_SIDECAR_URL.

Domain shows a different project (VMP CRM installer, cPanel, etc.)

An old nginx server-block still owns the domain. Return to section 7.0 and disable it. If HTTPS still shows the old page after fixing nginx, certbot bound the cert to the wrong config — reissue it:

sudo certbot delete --cert-name wa.animitra.in
sudo certbot --nginx -d wa.animitra.in \
             --agree-tos -m admin@animitra.in \
             --redirect --non-interactive

502 from Nginx

9 times out of 10 the backend crashed. Check supervisorctl status wa_api_backend, tail the log, fix, restart.
